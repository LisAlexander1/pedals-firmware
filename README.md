# Pedals Firmware (Arduino Pro Micro)

Прошивка симрейсингового контроллера педалей на базе **Arduino Pro Micro**
(ATmega32U4). Устройство определяется в системе как **HID-джойстик с тремя
осями** (Газ / Сцепление / Тормоз) и одновременно как **Serial-устройство**
с простым текстовым протоколом для калибровки и настройки.

Веб-приложение для калибровки лежит в соседнем репозитории
[`pedals-webapp/`](../pedals-webapp/) — оно подключается по Web Serial и
умеет даже прошивать эту плату прямо из браузера.

## Возможности

- **3 оси HID** через [ArduinoJoystickLibrary](https://github.com/MHeironimus/ArduinoJoystickLibrary)
  — Accelerator (газ), Brake (тормоз), Steering (сцепление), диапазон
  `0..1023`.
- **2 датчика Холла Honeywell S49E** на аналоговых входах — для газа и
  сцепления.
- **Тензодатчик через HX711** (24-бит АЦП) — для тормоза. Чтение
  неблокирующее (`is_ready()`), не тормозит остальные оси.
- **Калибровка** диапазона `min` / `max` каждой оси одной командой.
- **Мёртвые зоны** `low` / `high` с растяжением остального участка.
- **Кривая отклика** — 6 точек, кусочно-линейная интерполяция.
- **Фильтр шумов** на ось: `none`, EMA (экспоненциальное скользящее
  среднее), SMA (простое скользящее по N отсчётов).
- **Инверсия оси** программно, без перепайки.
- **Тара** тензодатчика тормоза.
- **EEPROM**: вся конфигурация хранится в EEPROM и автосохраняется при
  каждом изменении (через `EEPROM.put`, который под капотом
  `EEPROM.update` и не перезаписывает неизменившиеся байты).
- **Текстовый протокол** через USB-CDC — никакого JSON, никаких
  динамических аллокаций.

## Аппаратная схема

| Сигнал           | Пин Pro Micro | Примечание                                |
| ---------------- | ------------- | ----------------------------------------- |
| Throttle (S49E)  | A0            | аналоговый, 10-бит (0..1023)              |
| Clutch (S49E)    | A1            | аналоговый                                |
| HX711 DOUT       | D4            | data ready / data line                    |
| HX711 SCK        | D5            | clock                                     |
| S49E Vcc / GND   | 5V / GND      | S49E ratiometric относительно Vcc         |
| HX711 Vcc / GND  | 5V / GND      | модуль HX711 питается от 5 В              |

Распиновка вынесена в [`platformio.ini`](platformio.ini) через
`build_flags` — менять можно без правки исходников. Там же лежат:
скорость UART, диапазон выходных значений HID, gain HX711, дефолты
калибровки, размеры мёртвой зоны и т.д.

## Сборка

Проект под [PlatformIO](https://platformio.org/). Зависимости тянутся
автоматически:
- `robtillaart/HX711`
- `MHeironimus/ArduinoJoystickLibrary` (через git URL)

```bash
pio run            # компиляция
pio run -t upload  # прошить плату
pio device monitor # открыть Serial для отладки (115200 8N1)
```

В CI ([`.github/workflows/build.yml`](.github/workflows/build.yml))
каждый push в `main` собирает прошивку и публикует `firmware.hex` как
артефакт. Push тега `v*` дополнительно создаёт GitHub Release с
прикреплёнными `firmware.hex` / `firmware.elf` — webapp умеет
автоматически тянуть последний релиз и заливать в плату.

Текущий расход ATmega32U4:
- **Flash ≈ 49%** (≈ 14 КБ из 28 КБ)
- **RAM ≈ 37%** (≈ 940 байт из 2.5 КБ)

## Структура исходников

```
src/
  main.cpp         — setup/loop, экземпляры Joystick/HX711, опрос осей
  pedals.h/.cpp    — фильтр (EMA/SMA), нормализация, мёртвые зоны, кривая
  calibration.h/.cpp — структура конфигурации, EEPROM, дефолты
  protocol.h/.cpp  — построчный текстовый Serial-протокол
```

## Текстовый протокол по Serial

Скорость **115200 бод**, кодировка ASCII, каждая команда / ответ
заканчивается `\n`. Поля разделяются пробелом. JSON не используется —
это дёшево по Flash и RAM на AVR.

Полная спецификация — в шапке файла [`src/protocol.cpp`](src/protocol.cpp).

### Команды (PC → устройство)

| Команда                                   | Описание                              |
| ----------------------------------------- | ------------------------------------- |
| `HI`                                      | Handshake, ждём ответ `HELLO`         |
| `STAT`                                    | Снимок состояния (`S …`)              |
| `CAL`                                     | Полная калибровка блоком (`CAL … END`)|
| `STR <0\|1> [period_ms]`                  | Стрим состояния вкл/выкл              |
| `SAV`                                     | Принудительная запись в EEPROM        |
| `RST [<axis>]`                            | Сброс одной оси либо всех             |
| `MIN <axis>`                              | Текущее raw → min                     |
| `MAX <axis>`                              | Текущее raw → max                     |
| `DZ <axis> <low> <high>`                  | Мёртвые зоны 0..511                   |
| `INV <axis> <0\|1>`                       | Инверсия оси                          |
| `FLT <axis> <type:0..2> <strength:0..255>`| Фильтр (`0=none, 1=ema, 2=sma`)       |
| `CRV <axis> x0 y0 x1 y1 x2 y2 x3 y3 x4 y4 x5 y5` | Кривая (6 точек)               |
| `TAR <axis>`                              | Тара тензодатчика (только axis=2)     |

`axis`: `0 = throttle`, `1 = clutch`, `2 = brake`.

### Ответы (устройство → PC)

| Префикс | Формат                                                  |
| ------- | ------------------------------------------------------- |
| `HELLO` | `HELLO pedals <CALIB_VERSION>` — магия идентификации    |
| `OK`    | `OK <cmd>` — успех                                      |
| `ER`    | `ER <cmd> <reason>` — ошибка                            |
| `S`     | `S <ms> <r0> <r1> <r2> <o0> <o1> <o2>` — снимок         |
| `CAL`   | начало блока калибровки                                 |
| `A`     | `A <axis> <min> <max> <dzL> <dzH> <inv> <ftype> <fstr> <raw> <out>` |
| `C`     | `C <axis> x0 y0 … x5 y5` — кривая                       |
| `END`   | конец блока калибровки                                  |

### Пример сессии

```
→ HI
← HELLO pedals 1
→ CAL
← CAL
← A 0 120 880 10 10 0 1 96 134 8
← C 0 0 0 204 204 409 409 614 614 819 819 1023 1023
← A 1 …
← C 1 …
← A 2 …
← C 2 …
← END
→ STR 1 30                    # стрим со скоростью ~33 Гц
← OK STR
← S 12345 134 56 0 8 2 0
← S 12378 145 56 1 14 2 1
…
→ MIN 0                       # отпущена педаль газа → текущее raw в min
← OK MIN
```

## Алгоритм обработки оси

```
raw  ─►  filter  ─►  clamp(min..max) + map → 0..1023  ─►  deadzone  ─►  curve  ─►  HID
```

1. **Filter** (на ось): `none` / `EMA(α = strength/255)` / `SMA(N = strength)`.
2. **Inversion + linear scale**: `[rawMin..rawMax] → [0..1023]`. При
   `inverted=true` границы свапаются.
3. **Deadzones**: всё что `≤ dzLow` → 0, всё что `≥ 1023-dzHigh` → 1023,
   оставшийся отрезок растягивается обратно на 0..1023.
4. **Curve**: бинарный поиск отрезка кривой, кусочно-линейная
   интерполяция между точками.

## EEPROM

Структура `Calibration` хранится по адресу 0 с magic-числом
`0xC0FFEE02`. При несовпадении magic / версии — заполняется дефолтами
(см. `calibration_reset_*` в [`src/calibration.cpp`](src/calibration.cpp)).

Запись через `EEPROM.put()` использует `update()` — байты, которые не
изменились, не перезаписываются. Это снижает износ ячеек при частой
калибровке.
